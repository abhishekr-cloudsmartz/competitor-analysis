All steps are automated and serverless — runs on GitHub's free tier with no infrastructure to manage.

⏰ When does it run?

  • Automatically — GitHub Actions cron fires every weekday at 10:30 AM IST. The Gemini 2.5 Flash model generates the brief from RSS data.
  • Manually — Ask Claude Code to "run the brief". This picks up any signals that arrived after the morning run, without duplicating the day's entry.
  • Multi-run safe — Each run compares new RSS links against the stored rssSnapshot. If nothing is new, it exits without writing. Today's entry is replaced (not duplicated) when new signals arrive.

🔍 What does it track?

  • 12 competitors across 3 tiers (Enterprise, Mid-Market, Peer) — configured in config.json.
  • Google Alerts RSS feeds — one per competitor. Surfaces press releases, news articles, and occasional LinkedIn posts.
  • LinkedIn signal count — RSS items sourced from linkedin.com are counted separately as a posting-frequency indicator.
  • 48-hour delta only — not a full archive scan. Each day's brief is incremental: what changed since yesterday.

🗺️ How to use this dashboard

  • Overview — Channel benchmarks, LinkedIn follower rankings, event presence matrix. Static intelligence — updated manually when the competitive landscape shifts.
  • Activity — Daily briefs. Click any date in the sidebar to see that day's signals, top finding, channel pulse, and suggested action.
  • Competitor tabs (Peer / Mid-Market / Enterprise) — Deep profiles: products, channel strategy, recent signals, and positioning vs CloudSmartz.
  • Event matrix — Which competitor attended which conference. Click ◆ to open the source evidence in a new tab.

🛠️ Adding competitors or events

  • All competitor data lives in config.json — add a new entry there and the nav, matrices, and tables update automatically.
  • Set up a Google Alert for the new competitor at google.com/alerts, subscribe via RSS, and paste the feed URL into googleAlertRssUrl.
  • Events are defined in the top-level config.json → events array. Add an entry and reference the ID in the competitor's eventAttendance object.
  • The GitHub repo is private. The dashboard is served via GitHub Pages from the main branch.
